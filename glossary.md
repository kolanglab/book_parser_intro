# 用語集

本書で登場した主要な用語を、初めて出会った読者にも分かるよう簡潔にまとめます。各用語には、本書のどの章で詳しく扱ったかを添えてあります。読み返しの索引として、また知識の整理にお使いください。

## 解析の基本概念

**字句解析（lexical analysis / lexing）**
文字の並びを、意味のある最小単位であるトークンへまとめる処理。空白の読み飛ばしや、複数文字からなる数値・識別子のまとめ上げを担う。正規表現で表せる範囲（正規言語）に対応する。第1章で導入。

**トークン（token）**
字句解析が出力する、意味の最小単位。「整数」「識別子」「プラス記号」といった**種別**と、`23` や `x` といった具体的な**字句（lexeme）**を持つ。構文解析の入力となる。第1章。

**構文解析（parsing）**
トークン列を文法に従って木構造へ組み上げる処理。本書全体の主題。トップダウンとボトムアップという二つの戦略がある。第1章で導入。

**抽象構文木（Abstract Syntax Tree / AST）**
プログラムの構造を、意味のある要素だけを残した木として表現したもの。文法の都合による中継ぎノードを含む**具象構文木（構文木 / parse tree）**と対比される。後段の評価・コード生成・静的解析すべての土台。第1章・第2章。

## 文法の理論

**文脈自由文法（Context-Free Grammar / CFG）**
非終端記号をその前後の文脈に関係なく展開できる文法。終端記号・非終端記号・生成規則・開始記号からなり、プログラミング言語の再帰的な構文の記述に適する。第2章。

**終端記号 / 非終端記号（terminal / nonterminal）**
終端記号は入力にそのまま現れる記号（トークンに対応）。非終端記号は規則でさらに展開される「構造の名前」（`expr` など）。第2章。

**BNF / EBNF**
文法を規則の集まりとして書き表す記法。`::=` で定義、`|` で選択肢を表す。EBNF は繰り返し（`*`）や省略可能（`?`）を加えた拡張版。第2章。

**導出（derivation）**
開始記号から規則を繰り返し適用して、ある文字列を作り出す手続き。構文解析はこの導出を逆向きにたどる作業に相当する。第2章。

**曖昧性（ambiguity）**
一つの文字列に対し、構造の異なる構文木が2つ以上存在する文法の性質。優先順位を文法に埋め込むか、優先順位宣言で解消する。「ぶら下がり else」が典型例。第2章。

**正規言語（regular language）**
正規表現や有限オートマトンで表せる言語のクラス。括弧の対応のような「無制限に数える」処理はできず、これが字句解析と構文解析を理論的に分ける理由となる。第6章。

## 解析手法

**再帰下降構文解析（recursive descent parsing）**
非終端記号ごとに関数を一つ用意し、関数の相互再帰で入れ子構造を、呼び出し階層で優先順位を実現するトップダウン手法。手書きで実装され、多くの本物の処理系が採用する。第3章。

**左再帰（left recursion）**
規則の左端に自分自身が現れること（`expr ::= expr "+" term`）。トップダウン（LL）方式では無限再帰を招くため書き換えが必要だが、ボトムアップ（LR）方式は自然に扱える。第3章・第6章。

**LL 構文解析**
入力を左から読み、左端の規則から導出するトップダウン手法。`k` トークンを先読みするものを LL(k) と呼ぶ。FIRST 集合・FOLLOW 集合をもとに「どの規則を使うか」を予測する。第6章。

**LR 構文解析**
入力を左から読み、右端の規則から導出するボトムアップ手法。トークンを積む**シフト**と、規則でまとめる**還元**を繰り返す。左再帰を自然に扱える。Knuth が1965年に確立。第6章。

**LALR(1)**
LR(1) の状態を併合して表を小さく保ちつつ、近い表現力を得る実用的な手法。yacc／Bison が採用。DeRemer が博士論文で導入。第6章。

**コンフリクト（conflict）**
LR 系ジェネレータで、シフトと還元のどちらを行うべきか（シフト／還元コンフリクト）、あるいはどの規則で還元すべきか（還元／還元コンフリクト）を一意に決められない状況。文法の曖昧さや手法の限界の表れ。第4章・第6章。

**PEG（Parsing Expression Grammar / 解析表現文法）**
順序付き選択（`/`、左から試して最初に成功したものを採る）により曖昧性が原理的に生じない文法形式。Bryan Ford が2004年に提案。第5章・第6章。

**パックラット構文解析（packrat parsing）**
「位置×規則」の解析結果をすべてメモ化することで、無制限の先読みを持つ PEG を線形時間で実行する技法。メモリと引き換えに速度を得る。第5章。

**Earley 法**
左再帰や曖昧さを含むあらゆる文脈自由文法を解析できる汎用手法。一般には `O(n³)` と重いが、素直な文法では高速化する。第6章。

**Pratt 構文解析（演算子順位解析）**
演算子に「束縛力」という数値を与え、式の優先順位を単一の関数でスマートに扱う実用テクニック。再帰下降の親戚。第3章・第6章。

## 道具とシステム

**パーサージェネレータ（parser generator）**
文法定義を入力すると、それを解析するパーサのソースコードを自動生成する道具。yacc／Bison、ANTLR などがある。文法を唯一の真実とする。第4章。

**意味アクション（semantic action）**
パーサージェネレータの文法中に書く、規則が認識されたときに実行されるコード。AST のノード構築などを行う。第4章。

**ALL(\*)（Adaptive LL(\*)）**
ANTLR 4 が採用する、規則選択を実行時の動的解析で決めるトップダウン手法。左再帰を自動処理し、固定先読みでは区別できない文法も扱う。第5章・第6章。

**パーサーコンビネータ（parser combinator）**
専用ツールを使わず、ホスト言語の関数とその合成（連接・選択・繰り返しなどのコンビネータ）だけでパーサを組み立てる手法。第5章。

**一般化構文解析（generalized parsing）**
LR／LL の効率を保ちつつ、あらゆる CFG を扱う手法群。競合時に状態を分裂させて並行解析する **GLR**（富田）と、LL を一般化した **GLL**（Scott と Johnstone）がある。状態爆発は GSS で抑える。第7章。

**微分による構文解析（parsing with derivatives）**
1964年の「正規表現の微分」を CFG に拡張した、簡潔で数学的に美しい解析手法。左再帰も曖昧さも自然に扱う。第7章。

**インクリメンタル構文解析（incremental parsing）**
変更箇所だけを再解析して構文木を更新する手法。エディタや IDE の編集に即応するため不可欠。tree-sitter が代表的な実用システム。第7章。

**エラー回復（error recovery）**
構文エラーで止まらず、後続のエラーもまとめて報告したり、不完全なコードからもっともらしい木を返したりする技術。現代の処理系の使い勝手を左右する。第7章。

**スキャナレス構文解析（scannerless parsing）**
独立した字句解析器を置かず、文字を直接パーサに食わせる方式。言語の埋め込みや合成、文脈依存の字句を扱いやすい。第7章。
